🛡️ Security Model

Designed for paranoia.

Symmetric-only cryptography. No public keys to steal. Post-quantum safe by construction.

1

Master Password

Entered interactively or via environment for CI. Never written to disk.

2

Argon2id Key Derivation

Password + 256-bit salt → 256-bit AES key. Memory-hard and GPU-resistant.

3

AES-256-GCM Encryption

Fresh random 96-bit nonce on every save. Authenticated — tampered data is rejected.

4

Atomic Persistence

Write to temp → rename. A crash mid-save never corrupts the vault file.

5

Memory Zeroization

AES key and secret values are overwritten with zeros on drop.

🔐 Vault File Structure

Only the format version, creation timestamp, and Argon2id salt are stored in the clear. Key names, values, descriptions, tags, and the entire audit log are encrypted within the AES-256-GCM ciphertext.

🔄 Password Rotation

heist rotate generates a new salt, derives a new key, and re-encrypts the entire vault — secrets and audit log — in a single atomic operation. The old password is immediately invalidated.

⌨️ Commands

Simple by design.
Powerful by default.

Intuitive subcommands, shell completions, and scriptable output.

Basics
# Create a new vault
$ heist init
 
# Store a secret (interactive hidden prompt)
$ heist set github/token --tags "github,prod"
 
# Retrieve it
$ heist get github/token
ghp_abc123...
 
# Copy to clipboard (auto-clears in 45s)
$ heist copy github/token
✓ Copied to clipboard. Clears in 45 seconds.
Manage Secrets
# List all secrets
$ heist list
 
# Filter by namespace
$ heist list aws/
 
# Filter by tag
$ heist list --tag prod --tag db
 
# Machine-readable output
$ heist list --json | jq '.[] | .value'
 
# Delete a secret
$ heist rm old/api-key --yes
Workflow
# Inject secrets into a subprocess
$ heist exec aws/access-key aws/secret-key -- terraform apply
 
# Import from .env file
$ heist import .env --namespace prod
✓ Imported 12 secrets (0 skipped).
 
# Export as JSON
$ heist export aws/ --format json -o aws-secrets.json
 
# Rotate master password
$ heist rotate
✓ Master password rotated. Vault re-encrypted with new key.
CI / Automation
# Password via file (recommended for CI)
$ export HEIST_PASSWORD_FILE=/run/secrets/heist
 
# Or via environment variable
$ export HEIST_MASTER_PASSWORD="$CI_SECRET"
 
# Non-interactive — no TTY needed
$ heist exec DEPLOY_TOKEN -- ./deploy.sh
 
# Multiple vaults for different environments
$ heist --vault staging.heist list
$ heist --vault production.heist list
📦 Installation

Up and running in 30 seconds.

Single static binary. No config. No daemon. No internet access required at runtime.

📥

Pre-built Binary

Download for macOS, Linux, or Windows from the releases page.

curl -Lo heist https://github.com/iamdainwi/heist/releases/latest/download/heist
📦

Cargo Install

Install from crates.io with the Rust toolchain.

cargo install heist
🔧

Build from Source

Clone and build with Rust 1.74+.

git clone https://github.com/iamdainwi/heist && cd heist && cargo build --release
⚛️ Post-Quantum

Quantum resistant
by construction.

heist uses only symmetric cryptography. Shor's algorithm — the quantum threat to public-key crypto — has nothing to attack. Grover's algorithm reduces AES-256 to an effective 128-bit security level, which NIST considers sufficient.

Primitive Quantum Threat Post-Quantum
AES-256-GCM Grover (quadratic) ~128-bit ✓
Argon2id (256-bit) Grover (quadratic) ~128-bit ✓
Public-key crypto Shor (exponential) Not used ✓
